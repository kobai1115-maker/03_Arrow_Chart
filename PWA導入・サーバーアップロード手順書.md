# アローチャートアプリ PWA導入・サーバーアップロード手順書

本ドキュメントでは、アローチャートアプリをさまざまな端末（Windows、macOS、Linux、iPad、Androidタブレット）でアプリとしてインストールして利用するための「PWA（Progressive Web App）化」および「サーバーへのアップロード方法」について解説します。

---

## 1. アプリのビルド（Web用データの出力）

サーバーにアップロードする前に、FlutterのソースコードからWebブラウザで動作するHTML/CSS/JavaScriptなどのデータを作成（ビルド）する必要があります。

### 手順
1. コマンドプロンプトまたはPowerShell等のターミナルを開き、アプリのルートディレクトリ（`アローチャート`フォルダ）に移動します。
2. 以下のコマンドを実行します。
   ```bash
   flutter build web --release
   ```
3. 少し待つとビルドが完了します。
4. アプリのフォルダ内に `build/web/` というフォルダが作成され、その中に生成されたすべてのファイル（`index.html`など）が格納されます。**この `web` フォルダの中身すべてが、後でサーバーにアップロードする対象となります。**

---

## 2. サーバーへのアップロード方法

Web用データを公開するためには、インターネット上のサーバーへアップロードする必要があります。PWAとして動作させるには、**HTTPS対応（SSL化）** が必須です。

用途や現在の環境に応じて、主に以下の2パターンの方法があります。

### パターンA：一般的なレンタルサーバーを利用する方法
法人や個人ですでにWebサイト用のサーバー（エックスサーバー、さくらのレンタルサーバ、ロリポップなど）を契約している場合におすすめです。

#### 必要なもの
- レンタルサーバーの契約（独自ドメインおよびSSL設定済み）
- FTPクライアントソフト（FFFTP、FileZilla、WinSCPなど）

#### 手順
1. FTPクライアントソフトを使用して、契約しているレンタルサーバーへ接続します。（接続に必要なホスト名、ユーザー名、パスワードはサーバー会社の管理画面等で確認できます）
2. 公開したいフォルダ（例：`public_html` 直下、あるいは `public_html/arrowchart/` など用途に合わせたディレクトリ）を開きます。
3. ローカルPCの `build/web/` フォルダの**中身すべて**（`index.html` や `assets` フォルダなど）を、FTPソフトを使ってサーバーの公開フォルダへドラッグ＆ドロップでアップロードします。
4. アップロード完了後、ブラウザで配置したURL（例：`https://あなたのドメイン.com/arrowchart/`）にアクセスし、アプリが表示されるか確認します。

### パターンB：無料のホスティングサービスを利用する方法（例：Firebase Hosting）
サーバーをお持ちでない場合、Googleが提供する `Firebase Hosting` 等を利用すると、無料で高速・安全（HTTPS標準対応）な環境を構築できます。

#### 必要なもの
- Googleアカウント
- Node.js がインストールされていること（Firebase CLI を使うため）

#### 手順
1. **Firebaseプロジェクトの作成**
   - [Firebase Console](https://console.firebase.google.com/) にアクセスし、Googleアカウントでログインして「プロジェクトを追加」から新しいプロジェクトを作成します。
2. **Firebase CLIのインストール（初回のみ）**
   - コマンドプロンプトで以下を実行します：`npm install -g firebase-tools`
   - 続いて `firebase login` を実行し、ブラウザでGoogleアカウントを認証します。
3. **プロジェクトの初期化**
   - アローチャートアプリのルートフォルダでコマンドプロンプトを開き、`firebase init hosting` を実行します。
   - プロジェクトを選択し、「What do you want to use as your public directory?」と聞かれたら `build/web` と入力します。
   - 「Configure as a single-page app (rewrite all urls to /index.html)?」は `Yes` にします。
4. **アップロード（デプロイ）**
   - `firebase deploy --only hosting` を実行します。
   - 完了すると、公開用のURL（`https://プロジェクト名.web.app` など）が表示されます。

## 3. 各端末へのインストール（PWAとしての追加）

サーバーへのアップロードが完了し、指定のURLにアクセスできるようになったら、各端末で以下の操作を行うことでアプリとしてインストールできます。

### Windows / macOS / Linux (PC)
1. **Google Chrome** または **Microsoft Edge** で公開URLにアクセスします。
2. アドレスバーの右端（ブックマークアイコン付近）に表示される「**アプリをインストール**」アイコン、または「インストール」等のボタンをクリックします。
3. 確認画面が出たら「インストール」を選択します。
4. PCのアプリケーションとして登録され、デスクトップのショートカットやスタートメニュー/Launchpadから直接起動できるようになります。

### iPad (iOS / iPadOS)
1. **Safari** を開いて、公開URLにアクセスします。（※Chrome等他のブラウザでは追加できません）
2. 画面上部（または下部）にある「共有」ボタン（四角から上向き矢印が出ているアイコン）をタップします。
3. メニューをスクロールして「**ホーム画面に追加**」をタップします。
4. 右上の「追加」をタップすると、ホーム画面にアプリアイコンが作成されます。

### Android タブレット (スマートフォン含む)
1. **Google Chrome** を開いて、公開URLにアクセスします。
2. 画面下部に「ホーム画面に追加」のポップアップが出た場合はそれをタップします。
3. 出ない場合は、ブラウザ右上の「︙（3点リーダーメニュー）」を開き、「**アプリをインストール**」または「**ホーム画面に追加**」をタップします。
4. ダイアログに従って追加すると、ホーム画面にアプリアイコンが作成されます。

## 4. アプリ更新時の手順
アプリのプログラムを修正・アップデートした場合は、以下の手順でサーバーのデータを差し替えます。

1. 再度 `flutter build web --release` を実行して、最新のWebデータを生成します。
2. FTPソフト等でサーバーの古いファイルを上書きアップロードする、または `firebase deploy` 等でデプロイを実行します。
3. **注意点**: 利用者の端末では、ブラウザのキャッシュ（一時データ）の都合で古いバージョンが表示されることがあります。アプリ起動時に自動更新される仕組みがない場合は、ブラウザでページをスーパーリロード（キャッシュクリア再読み込み）するか、アプリを一度削除して再インストールしてもらう必要があります。（※PWAのServiceWorker設定を調整することで自動更新させることも可能です）
