# 【初心者向け】FlutterアプリをGitHubに公開するための完全手順書

この手順書では、あなたが開発した「アローチャート・アプリ」を、世界中のプログラミング・ソースコードが保管されている場所（GitHub）へアップロードする方法を、一歩ずつ丁寧に解説します。

---

## 1. アプリの名前（リポジトリ名）を決めよう

GitHubでは、プロジェクトを保存する場所を「リポジトリ」と呼びます。
今回開発しているアローチャート・アプリにふさわしい名前の候補をいくつか提案します。

- **候補1：`arrow-chart-care`**
  - **理由**: 「アローチャート」と「ケア（介護）」というキーワードが組み合わさっており、どんなアプリか一目でわかります。
- **候補2：`care-manager-arrow-tool`**
  - **理由**: ケアマネジャーさん向けのツールであることを強調しており、ターゲット層にアピールしやすい名前です。
- **候補3：`nursing-arrow-chart`**
  - **理由**: 海外のエンジニアにも「介護・看護（Nursing）」分野のツールであることがすぐに伝わります。

**💡 命名のコツ（命名規則）**
- **英単語をハイフン(`-`)でつなぐ**: これがエンジニアの間で最も一般的な書き方です（例: `my-awesome-app`）。
- **すべて小文字**: GitHubでは、小文字で統一するのが一般的です。

### リポジトリの「説明 (Description)」の案
GitHubには「Description（概要）」という、そのアプリが何をするものかを短く書く欄があります。ここに入力するおすすめの文章をいくつか用意しました。

- **案1（シンプル）**:
  - `介護・看護のケアプラン作成を支援するアローチャート作成ツール`
- **案2（対象を明示）**:
  - `ケアマネジャー向けの介護福祉アローチャート支援アプリ`
- **案3（英語も併記：おすすめ！）**:
  - `介護・看護のためのアローチャート作成支援ツール / Arrow Chart assistant for nursing and welfare.`

※Descriptionは後からでも変更できるので、まずは案1のような日本語で入力しておけば大丈夫です！

---

## 2. 覚えておきたい！魔法の言葉（用語解説）

手順を進める前に、よく出てくる言葉を「例え話」で理解しておきましょう。

- **Git（ギット）**:
  - ファイルの「変更履歴を記録する機械」です。過去の状態に戻したり、誰がいつ何を変えたか記録したりできます。
- **GitHub（ギットハブ）**:
  - Gitで記録したファイルを保存しておく「ネット上の倉庫」です。
- **コマンド**:
  - パソコンに魔法の呪文を唱えて命令することです。VS Codeの「ターミナル（黒い画面）」に入力します。
- **リポジトリ**:
  - ファイルを保管する「本棚」や「倉庫」のことです。自分のPC内にあるのを「ローカルリポジトリ」、ネット上にあるのを「リモートリポジトリ」と呼びます。
- **.gitignore（ギットイグノア）**:
  - 「これは倉庫に入れなくていいよ」と指示する「除外リスト」のことです。
- **コミット (Commit)**:
  - 変更した内容を「確定して名前を付ける」作業です。セーブポイントを作るイメージです。
- **プッシュ (Push)**:
  - 自分のPCにあるファイルを、ネット上の倉庫（GitHub）へ「送り届ける」ことです。
- **ブランチ (Branch)**:
  - 「枝分かれ」のことです。本番用の枝、実験用の枝、といった風に作業を分けることができます。最初は「main（メイン）」という枝を使います。

## 3. 事前準備（パソコンの準備）

まずは、道具を揃えましょう。

1.  **Git for Windowsのインストール**:
    - [Git for Windows 公式サイト](https://gitforwindows.org/) に行き、ダウンロードしてインストールしてください。設定はすべて「Next」のままで大丈夫です。
    - **【重要】インストール直後にブラウザが開いた場合**:
      - 自動的に「Release Notes（リリースノート）」という英語の画面が表示されることがありますが、これは単なる新機能の紹介です。**内容は読まずにそのまま閉じてOKです！**
2.  **GitHubアカウントの作成**:
    - [GitHub公式サイト](https://github.com/) でアカウントを作っておいてください。

---

## 4. VS Codeで「アローチャート」を開く

1.  VS Codeを立ち上げます。
2.  「ファイル」メニュー ＞ **「フォルダーを開く」** で、アローチャートのソースコードが入っているフォルダ（`arrow_chart_app`）を選択します。
3.  メニューの「表示」 ＞ **「ターミナル」** をクリックします。画面の下半分に黒い（または青い）画面が出てきます。

---

## 5. Gitの初期設定（あなたの名前を登録）

ここでよくある疑問にお答えします。**「Gitというアプリを新しく立ち上げるの？」** と思われるかもしれませんが、その必要はありません。

- **Gitは「エンジン」です**: 車のエンジンのように、裏側で動く仕組みです。
- **操作は「ターミナル」で**: Step 4で開いた **VS Codeの「ターミナル（黒い画面）」** が、Gitに命令を伝えるための「ハンドル」や「アクセル」になります。

では、パソコンに「この変更をしているのは私（あなた）です」と教える設定をしていきましょう。

### 5-1. なぜ設定が必要なの？
Gitは「誰が・いつ・何を」変えたかを記録する道具です。ここで設定した名前とメールアドレスは、GitHub上の履歴（いつ誰が保存したか）として表示されるようになります。

### 5-2. 設定コマンドの実行
ターミナルに以下のコマンドを **1行ずつ** コピーして貼り付け、Enterキーを押してください。

```bash
# 名前を登録（your_name の部分を半角英数字の名前に変えてください）
# 例: git config --global user.name "taro-yamada"
git config --global user.name "your_name"

# メールアドレスを登録（GitHubに登録したものと同じメールアドレスにしてください）
# 例: git config --global user.email "taro@example.com"
git config --global user.email "your_email@example.com"
```

**💡 名前はどう決めればいいの？（よくある質問）**
- **GitHubのユーザー名と揃えるのが一番オススメです！**
  - そうすることで、「GitHub上のあなた」と「手元のPCのあなた」が一致し、後で変更履歴を見たときに誰の作業か分かりやすくなります。
- **一度設定すればOK！**
  - この設定はパソコン全体で一度だけ行えば大丈夫です（`--global` という指定がその意味です）。プロジェクトごとに毎回考え直す必要はありません。

**⚠️ 注意点**
- **半角英数字**で入力してください（全角文字だとエラーになることがあります）。
- 名前はニックネームでも本名でも構いませんが、GitHubのユーザー名と揃えておくと分かりやすいです。

### 5-3. 正しく設定できたか確認する
入力が終わったら、以下のコマンドで正しく登録されたか確認してみましょう。

```bash
git config --list
```
ずらっと文字が出てきますが、その中に `user.name=あなたの名前` と `user.email=あなたのメールアドレス` が表示されていれば完璧です！
（確認が終わったら、ターミナルで `q` キーを押すと元の画面に戻れます）

## 6. ローカルリポジトリの作成 (`git init`)

さて、ここからが本番です！今のフォルダを「Gitで記録を開始する場所」として宣言します。

ターミナルに入力：
```bash
git init
```
※「Initialized empty Git repository...」と出れば成功です！

---

## 7. 公開しないファイルの設定 (`.gitignore`)

プログラムを動かすための一時的なファイルや、パスワードなどはGitHubにアップロードしてはいけません。
Flutterプロジェクトの場合、すでに `.gitignore` というファイルが初期状態で作られているはずですが、中身を確認しましょう。

1.  左側のファイル一覧から `.gitignore` をクリックして開きます。
2.  以下の内容が含まれているか確認してください（Flutter用の基本セットです）。

```text
# 不要なファイルを除外する魔法のリスト
.dart_tool/
.flutter-plugins
.flutter-plugins-dependencies
.packages
build/
ios/Flutter/Generated.xcconfig
android/key.properties
```

---

## 8. ファイルの追加と変更の記録 (`git commit`)

現在のファイルをすべて「セーブ」します。

1.  **「追加」のコマンド（倉庫に入れる準備）**:
    ```bash
    git add .
    ```
    ※末尾の `.` (ドット) は「このフォルダの全部！」という意味です。

2.  **「コミット」のコマンド（セーブポイント作成）**:
    ```bash
    git commit -m "最初のアップロード（Initial commit）"
    ```
    ※ `-m` の後ろに、どんな変更をしたかのメモを書きます。

---

## 9. GitHub上で「リモートリポジトリ」を作成する

いよいよネット上に倉庫を作ります。提供していただいた画面に沿って設定していきましょう。

1.  ブラウザで [GitHub](https://github.com/) にログインします。
2.  右上の「＋」ボタンをクリック ＞ **「New repository」** を選択。
3.  **【重要】設定値は以下のように入力してください：**

#### ① リポジトリ名 (Repository name)
- **入力**: `arrow-chart-care` (など)
- **理由**: 名前がないと作成できません。

#### ② 公開設定 (Visibility)
- **オススメ**: **「Private（自分だけ）」** を選択。
- **理由**: 「今はまだ世界中に公開したくない」「自分の控えとして使いたい」場合はこちらを選びます。後からいつでも公開設定（Public）に変更できるので、まずはPrivateで始めるのが最も安全です。

#### ③ 初期化設定 (Initialize this repository with...)
- **オススメ**: **すべて「チェックなし（None）」にする**。
- **理由（ここが一番大事です！）**:
  - すでに自分のPCにアプリのファイル（`.gitignore` など）が揃っているため、GitHub側で新しく自動作成してしまうと、**自分のPCにあるファイルと衝突（ぶつかってエラー）** してしまいます。
  - 後からスムーズにアップロードするために、ここは**「何も追加しない（空っぽの状態）」** で作成するのが正解です。

4.  一番下の **「Create repository」** 緑色のボタンを押します。

---

## 10. ローカルとリモートを紐付け、アップロードする！

リポジトリ作成直後の「Quick setup」という画面には、魔法の呪文（コマンド）が並んでいます。
今回は **「...or push an existing repository from the command line」**（すでに手元にあるフォルダをアップロードする）という項目にある3行を順番に使います。

### 10-1. リモート（GitHub）の場所を登録する
パソコンに「アップロード先の倉庫はここだよ」と教えます。

```bash
git remote add origin https://github.com/kobari1115-maker/arrow-chart-care.git
```
※URLの部分は、ご自身の画面に表示されているものをそのままコピーしてください。

### 10-2. メインの枝（ブランチ）の名前を確定する
リポジトリの「中心となる太い枝」の名前を「main」に変更します。

```bash
git branch -M main
```

### 10-3. ついにアップロード（プッシュ）！
いよいよ、手元のファイルをネット上の倉庫へ送り届けます。

```bash
git push -u origin main
```

---

## 💡 この画面に出てくる「難しい言葉」の徹底解説

最後に、上のコマンドに出てきた言葉の意味を詳しく紐解いてみましょう。

- **origin（オリジン）**:
  - ネット上の倉庫（GitHub側）に付ける **「あだ名」** です。毎回「https://github...」と長い住所を入力するのは大変なので、Gitの世界では慣習としてこれを「origin（大元）」と名付けて呼びます。
- **main（メイン）**:
  - プログラムの **「本番用の枝（幹）」** の名前です。昔は `master` と呼ばれていましたが、現在は `main` と呼ぶのが世界標準になっています。
- **-M（エム）**:
  - 「名前を強制的に変える(Modify)」という合図です。
- **-u（ユー）**:
  - **「次回から楽をするための魔法」** です。一度このオプションを付けてプッシュすると、パソコンが「次からは `git push` と打つだけで、自動的に origin の main 枝に送ればいいんだな」と覚えてくれます（これを「Upstreamの設定」と呼びます）。
- **HTTPS URL（https://github.com/...）**:
  - GitHub上のあなたの倉庫の **「住所」** です。世界に一つだけの専用の送り先になります。
- **remote（リモート）**:
  - 「遠く（ネット上）にある」という意味です。自分のパソコン内は「local（ローカル）」、ネット上は「remote（リモート）」と呼び分けます。

## 11. 【番外編】GitHubの登録情報（名前やメール）を変えた時は？

運用していく中で、GitHubのユーザー名やメールアドレスを変更したくなった場合の対処法です。

### 11-1. メールアドレスを変更した場合
GitHubに登録したメールアドレスを変更した際は、PC側の設定も新しいものに書き換える必要があります。

**例：`axissports2021@gmail.com` から `info@axlink.jp` に変えた場合**
ターミナルで以下のコマンドを実行してください。

```bash
git config --global user.email "info@axlink.jp"
```

### 11-2. ユーザー名（アカウント名）を変更した場合
ユーザー名を変えると、「PC側の名前設定」と「ネット上の倉庫の住所」の両方を更新する必要があります。

1.  **PC側の名前設定を更新**:
    ```bash
    git config --global user.name "新しい名前"
    ```

2.  **ネット上の倉庫の住所（URL）を更新**:
    ユーザー名が変わると、リポジトリのURLも変わってしまいます。以下のコマンドで新しい住所を教え直してください。
    ```bash
    # 例：新しいユーザー名が "axlink-user" の場合
    git remote set-url origin https://github.com/axlink-user/arrow-chart-care.git
    ```

### 💡 アドバイス
これらの変更を行った後は、一度 `git push` を試してみて、正しくアップロードできるか確認することをおすすめします！
